= Diagram Generation Overview

== Purpose

LutaML provides production-ready SVG diagram generation from Enterprise Architect models with pixel-perfect EA-style rendering.

== What is Diagram Generation?

Diagram generation converts UML diagram definitions from EA models into standalone SVG (Scalable Vector Graphics) files that match EA's visual output exactly.

== Key Features

=== EA-Style Rendering

Pixel-perfect match to Enterprise Architect's visual output:

* **Colors**: DataType (pink), FeatureType (yellow), GML types (green)
* **Fonts**: Calibri/Carlito matching EA defaults
* **Layout**: Exact element positioning from EA
* **Connectors**: All EA connector types supported
* **Stereotypes**: Rendered in EA format («stereotype»)

=== Configurable Styling

YAML-based configuration system:

* **Stereotype styling** - Colors by stereotype
* **Package patterns** - Styles by package hierarchy
* **Connector types** - Customize all connector styles
* **Font configuration** - Control all text rendering
* **Priority rules** - EA data > config > defaults

=== Multiple Element Types

Full support for UML elements:

* **Classes** - Three-compartment boxes
* **Packages** - Tab-style boxes
* **Data Types** - Structured types
* **Enumerations** - Literal lists
* **Interfaces** - Lollipop notation
* **Notes** - Documentation boxes

=== All Connector Types

Complete connector support:

* **Generalization** - Inheritance
* **Association** - Basic relationships
* **Aggregation** - Part-whole (hollow diamond)
* **Composition** - Ownership (filled diamond)
* **Dependency** - Usage relationships
* **Realization** - Interface implementation

=== Path Routing

Multiple routing algorithms:

* **Straight** - Direct lines
* **Manhattan** - Orthogonal routing
* **Orthogonal** - Right-angle connections
* **Bézier** - Curved connectors
* **Custom waypoints** - User-defined paths

== Architecture

=== Component Pipeline

[source]
----
Repository + Diagram ID
    ↓
DiagramPresenter (data extraction)
    ↓
LayoutEngine (coordinate conversion)
    ↓
StyleResolver (style merging)
    ↓
SvgRenderer (document assembly)
    ├→ ElementRenderers (class, package, etc.)
    └→ PathBuilder (connector paths)
    ↓
Complete SVG Document
----

=== Key Components

**DiagramPresenter**::
Extracts and transforms diagram data from repository

**LayoutEngine**::
Converts EA coordinates (left/top/right/bottom) to SVG (x/y/width/height)

**StyleResolver**::
Merges styles from EA data, user config, and defaults with priority

**SvgRenderer**::
Generates complete SVG with proper layering

**ElementRenderers**::
Render specific element types (classes, packages, etc.)

**PathBuilder**::
Parses EA geometry and builds SVG paths

== Using Diagram Generation

=== From QEA File

[source,ruby]
----
require 'lutaml'

# Load QEA file
document = Lutaml::Qea.parse("model.qea")
repo = Lutaml::UmlRepository::Repository.new(document: document)

# Get a diagram
diagram = repo.packages_index.flat_map(&:diagrams).first

# Generate SVG
presenter = Lutaml::UmlRepository::Presenters::DiagramPresenter.new(
  diagram,
  repo
)
svg = presenter.svg_output

# Save to file
File.write("diagram.svg", svg)
----

=== With Custom Styling

[source,ruby]
----
# Load custom configuration
presenter = Lutaml::UmlRepository::Presenters::DiagramPresenter.new(
  diagram,
  repo,
  config_path: "my_styles.yml"
)

# Generate with options
svg = presenter.svg_output(
  padding: 20,
  background_color: "#ffffff",
  grid_visible: false,
  interactive: false
)
----

=== Via CLI

[source,console]
----
# Extract single diagram
$ lutaml diagram extract model.qea -d EAID_12345 -o diagram.svg

# Extract all diagrams
$ lutaml diagram extract model.qea -o diagrams/

# List diagrams
$ lutaml diagram list model.qea
----

== Styling System

=== Three-Tier Priority

Styles merge from multiple sources:

1. **EA Data** (highest) - Colors/styles from .qea file
2. **User Configuration** - Custom YAML config
3. **Built-in Defaults** (lowest) - EA-style defaults

=== Configuration Example

[source,yaml]
----
stereotypes:
  DataType:
    colors:
      fill: "#FFCCFF"      # Pink
      stroke: "#000000"
    fonts:
      class_name:
        weight: 700
        style: "italic"

  FeatureType:
    colors:
      fill: "#FFFFCC"      # Yellow

packages:
  "CityGML::*":
    colors:
      fill: "#FFFFCC"

connectors:
  association:
    arrow:
      type: "open_arrow"
      size: 8
    line:
      stroke_width: 1
----

== Coordinate System

EA uses different coordinates than SVG:

[cols="1,2,2"]
|===
|System |Format |Example

|EA
|left/top/right/bottom
|`left: 100, top: 50, right: 200, bottom: 150`

|SVG
|x/y/width/height
|`x: 100, y: 50, width: 100, height: 100`
|===

LayoutEngine automatically converts between these systems.

== Output Formats

=== SVG (Primary)

Scalable vector graphics:

* Resolution independent
* Editable in vector tools
* Web-ready
* Small file size

=== PNG (Optional)

Raster graphics via `rsvg-convert`:

[source,console]
----
$ rsvg-convert -w 1920 diagram.svg > diagram.png
----

=== PDF (Optional)

Vector PDF via `inkscape` or `rsvg-convert`:

[source,console]
----
$ inkscape diagram.svg --export-pdf=diagram.pdf
----

== Performance

Typical performance characteristics:

* **Single diagram**: <1 second
* **Batch (100 diagrams)**: ~2 minutes
* **Memory**: Modest, processes one at a time
* **File size**: 10-100KB per diagram (SVG)

== Integration

=== With SPA Documentation

Diagrams automatically integrated:

[source,ruby]
----
# Generate documentation site with diagrams
repo.generate_static_site("output/",
  include_diagrams: true
)
----

=== With Custom Workflow

[source,ruby]
----
# Process all diagrams
repo.all_diagrams.each do |diagram|
  presenter = DiagramPresenter.new(diagram, repo)
  svg = presenter.svg_output

  # Save with diagram name
  File.write("diagrams/#{diagram.name}.svg", svg)
end
----

== Limitations

* **Read-only**: No editing of diagrams (use EA for that)
* **Layout approximate**: EA uses proprietary rendering engines
* **Diagram types**: Class diagrams fully supported, others partial
* **Custom shapes**: Standard UML shapes only
