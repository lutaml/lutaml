= EA Diagram Generation - Technical Guide
:toc:
:toclevels: 3

== Introduction

This document provides technical details on LutaML's EA-style diagram generation system. It covers architecture, implementation details, configuration, and extension patterns for developers working with or extending the diagram generation functionality.

== Architecture Overview

=== Component Hierarchy

[source]
----
Lutaml::UmlRepository::Presenters::DiagramPresenter
  • Entry point for diagram generation
  • Orchestrates the entire rendering pipeline
  • Transforms repository data to renderable format
  ↓
Lutaml::Ea::Diagram::LayoutEngine
  • Coordinate conversion (EA → SVG)
  • Bounds calculation
  • Auto-layout for unpositioned elements
  ↓
Lutaml::Ea::Diagram::StyleResolver
  • Multi-source style resolution
  • Configuration loading and caching
  • EA style string parsing
  ↓
Lutaml::Ea::Diagram::SvgRenderer
  • SVG document generation
  • Layer management
  • Renderer orchestration
  ↓
Lutaml::Ea::Diagram::ElementRenderers::*
  • BaseRenderer: Foundation for all renderers
  • ClassRenderer: Three-compartment class boxes
  • PackageRenderer: Tab-style package boxes
  ↓
Lutaml::Ea::Diagram::PathBuilder
  • EA geometry parsing
  • Connector path calculation
  • Waypoint handling
----

=== Component Responsibilities

==== DiagramPresenter

**Location**: `lib/lutaml/uml_repository/presenters/diagram_presenter.rb`

**Purpose**: Transforms EA diagram data from the repository into a format suitable for rendering.

**Key Responsibilities**:

* Extract elements from `diagram_objects`
* Extract connectors from `diagram_links`
* Resolve XMI IDs to actual UML elements
* Parse EA internal IDs (SOID, EOID, DUID)
* Extract element metadata (stereotypes, attributes, operations)
* Build source/target relationships for connectors

**Key Methods**:

* `svg_output(options)` - Main entry point for SVG generation
* `elements` - Returns array of element data hashes
* `connectors` - Returns array of connector data hashes
* `build_elements_data` - Private: Transforms diagram_objects
* `build_connectors_data` - Private: Transforms diagram_links

==== LayoutEngine

**Location**: `lib/lutaml/ea/diagram/layout_engine.rb`

**Purpose**: Handles coordinate conversion and layout calculation.

**Key Responsibilities**:

* Convert EA coordinates (left/top/right/bottom) to SVG (x/y/width/height)
* Calculate diagram bounds including connector endpoints
* Normalize coordinates to handle negative values
* Provide auto-layout for unpositioned elements

**Key Methods**:

* `convert_ea_coordinates(diagram_object)` - Convert single element
* `calculate_bounds(diagram_data)` - Calculate viewBox bounds
* `normalize_coordinates(elements)` - Ensure positive coordinates
* `apply_layout(elements, connectors)` - Auto-position elements

==== StyleResolver

**Location**: `lib/lutaml/ea/diagram/style_resolver.rb`

**Purpose**: Resolves styling from multiple sources with priority hierarchy.

**Key Responsibilities**:

* Load and cache configuration files
* Parse EA style strings (BCol, LCol, LWth, BFol, IFol)
* Resolve styles with priority: EA Data → Class → Package → Stereotype → Defaults
* Determine connector types (generalization, association, aggregation, composition)

**Key Methods**:

* `resolve_element_style(element, diagram_object)` - Get complete element style
* `resolve_connector_style(connector, diagram_link)` - Get complete connector style
* `resolve_fill_color(element, diagram_object)` - Get fill color
* `resolve_font(element, context)` - Get font properties

==== SvgRenderer

**Location**: `lib/lutaml/ea/diagram/svg_renderer.rb`

**Purpose**: Assembles complete SVG document from diagram data.

**Key Responsibilities**:

* Generate SVG header with viewBox
* Create defs section with CSS and markers
* Render layers in correct order (background, grid, connectors, elements)
* Determine correct markers for connector types
* Integrate element renderers and path builder

**Key Methods**:

* `render` - Generate complete SVG document
* `svg_header` - Generate SVG opening tag
* `defs_section` - Generate styles and markers
* `render_connector(connector)` - Render single connector
* `render_element(element)` - Render single element
* `determine_marker_type(type)` - Get marker URLs for connector type

==== Element Renderers

**Location**: `lib/lutaml/ea/diagram/element_renderers/`

===== BaseRenderer

**Purpose**: Foundation for all element renderers.

**Provides**:

* Basic shape rendering
* Label rendering with centering
* XML escaping for text content
* Style resolution integration

**Key Methods**:

* `render` - Main rendering method
* `render_shape(style)` - Abstract: Generate shape SVG
* `render_label(style)` - Abstract: Generate label SVG
* `escape_xml(text)` - Escape special characters

===== ClassRenderer

**Purpose**: Render UML class boxes with three compartments.

**Features**:

* Name compartment with optional stereotype
* Attributes compartment with visibility symbols
* Operations compartment with parameters and return types
* Dynamic height calculation
* Compartment separators

**Compartment Layout**:

[source]
----
┌─────────────────────────┐
│    <<Stereotype>>       │  ← Optional stereotype (italic)
│     ClassName           │  ← Class name (bold)
├─────────────────────────┤
│ + attr1: Type           │  ← Attributes with visibility
│ - attr2: Type           │
├─────────────────────────┤
│ + operation(): Type     │  ← Operations with parameters
│ # method(p: Type): Type │
└─────────────────────────┘
----

===== PackageRenderer

**Purpose**: Render UML package boxes with tab.

**Features**:

* Tab for package name (1/3 box width, 20px height)
* Main body for contents
* Separate styling for tab and body

**Package Layout**:

[source]
----
┌──────────┐
│ Package  │  ← Tab with name
└──────────┬─────────────────┐
           │                 │
           │    Body Area    │  ← Main content area
           │                 │
           └─────────────────┘
----

==== PathBuilder

**Location**: `lib/lutaml/ea/diagram/path_builder.rb`

**Purpose**: Generate SVG path commands from EA connector geometry.

**Key Responsibilities**:

* Parse EA geometry strings (SX/SY/EX/EY/EDGE/EDGEn)
* Calculate connection points on element edges
* Generate SVG path commands
* Support multiple routing: straight, manhattan, orthogonal, bezier

**EA Geometry Format**:

[source]
----
SX=10;SY=5;EX=-10;EY=-5;EDGE=1;EDGE1=250,125;EDGE2=300,175;
----

Where:

* `SX`, `SY` - Source offset from element center
* `EX`, `EY` - Target offset from element center
* `EDGE` - Edge identifier
* `EDGEn` - Waypoint coordinates (absolute)

== Coordinate Systems

=== EA Coordinate Format

Enterprise Architect stores element positions using a bounding-box format:

[source]
----
DiagramObject:
  left: 100      # X coordinate of left edge
  top: 50        # Y coordinate of top edge
  right: 220     # X coordinate of right edge
  bottom: 130    # Y coordinate of bottom edge
----

**Properties**:

* Origin at top-left (0, 0)
* Y-axis positive downward
* All values in pixels
* Can have negative values

=== SVG Coordinate Format

LutaML converts to standard SVG format:

[source]
----
Element:
  x: 100         # X coordinate of top-left corner
  y: 50          # Y coordinate of top-left corner
  width: 120     # Width of element (right - left)
  height: 80     # Height of element (bottom - top)
----

**Properties**:

* Origin at top-left (0, 0)
* Y-axis positive downward
* Matches CSS box model
* Normalized to positive values

=== Conversion Algorithm

[source,ruby]
----
def convert_ea_coordinates(diagram_object)
  {
    x: diagram_object.left,
    y: diagram_object.top,
    width: diagram_object.right - diagram_object.left,
    height: diagram_object.bottom - diagram_object.top
  }
end
----

=== ViewBox Calculation

The viewBox encompasses all elements and connectors with padding:

[source,ruby]
----
# Find minimum and maximum coordinates
min_x = elements.map { |e| e[:x] }.min
min_y = elements.map { |e| e[:y] }.min
max_x = elements.map { |e| e[:x] + e[:width] }.max
max_y = elements.map { |e| e[:y] + e[:height] }.max

# Include connector endpoints
connectors.each do |c|
  # ... extend bounds to include waypoints
end

# Apply padding
viewBox = "#{min_x - padding} #{min_y - padding} " \
          "#{max_x - min_x + 2*padding} #{max_y - min_y + 2*padding}"
----

== Connector Geometry

=== EA Geometry Format

EA stores connector geometry as a semicolon-delimited string:

[source]
----
SX=10;SY=5;EX=-10;EY=-5;EDGE=1;EDGE1=250,125;EDGE2=300,175;
----

**Components**:

`SX`, `SY`:: Source offset from element center (pixels)
`EX`, `EY`:: Target offset from element center (pixels)
`EDGE`:: Edge identifier (meaning varies by EA version)
`EDGEn`:: Waypoint n at absolute coordinates (x,y)

=== Waypoint Handling

PathBuilder processes waypoints in order:

[source,ruby]
----
# Parse geometry string
geometry.split(';').each do |pair|
  key, value = pair.split('=')
  if key =~ /^EDGE(\d+)$/
    # Waypoint found
    x, y = value.split(',').map(&:to_f)
    waypoints << { x: x, y: y }
  end
end
----

=== Path Generation

PathBuilder generates SVG path commands:

[source]
----
Start Point (element1 edge)
     │
     ▼
Waypoint 1 ──► Waypoint 2 ──► ... ──► Waypoint N
                                           │
                                           ▼
                                   End Point (element2 edge)
----

SVG Output:

[source,svg]
----
<path d="M 160,90 L 250,125 L 300,175 L 360,190" />
----

=== Configuration Loading

[source,ruby]
----
class Configuration
  def initialize(config_path = nil)
    @config = if config_path && File.exist?(config_path)
      YAML.load_file(config_path)
    else
      load_default_config
    end
  end

  def style_for(element, path)
    # Navigate nested hash using path (e.g., "colors.fill")
    keys = path.split('.')

    # Try class-specific
    if element.name && (class_config = @config.dig('classes', element.name))
      return navigate(class_config, keys) if navigate(class_config, keys)
    end

    # Try package wildcard
    if element.package_name
      @config.fetch('packages', {}).each do |pattern, config|
        next unless File.fnmatch(pattern, element.package_name)
        return navigate(config, keys) if navigate(config, keys)
      end
    end

    # Try stereotype
    if element.stereotype
      stereo_config = @config.dig('stereotypes', element.stereotype)
      return navigate(stereo_config, keys) if navigate(stereo_config, keys)
    end

    # Fall back to defaults
    navigate(@config['defaults'], keys)
  end
end
----

=== Priority Resolution Algorithm

[source]
----
For property P of element E:

1. IF EA data contains P
     RETURN EA value

2. IF class config for E.name contains P
     RETURN class config value

3. FOR EACH package pattern in config
     IF pattern matches E.package_name AND config contains P
       RETURN package config value

4. IF stereotype config for E.stereotype contains P
     RETURN stereotype config value

5. RETURN default value for P
----

== Troubleshooting

=== Common Issues

**Issue**: Elements not appearing in SVG

* Check XMI ID resolution in `find_element_by_xmi_id`
* Verify `diagram_objects` contains valid data
* Check coordinate values are not extreme

**Issue**: Connectors not connecting properly

* Verify source/target element resolution
* Check EA geometry string format
* Validate waypoint coordinates

**Issue**: Styles not applying

* Check configuration file syntax (valid YAML)
* Verify priority hierarchy (EA data overrides config)
* Check wildcard package matching

**Issue**: SVG not rendering

* Validate SVG with `Nokogiri::XML(svg) { |c| c.strict }`
* Check viewBox includes all elements
* Verify no XML special characters unescaped
